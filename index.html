MaisonLooks Public API

Handoff for static / build-time use · v1 · updated 2026-04-19

Important: Use the API only from server-side or build scripts. Do not put your API key in browser JavaScript or public HTML. Rate limit: 300 req/min per IP.

Authentication

  • Header: X-API-Key: <your key> (from env e.g. MATRIX_API_KEY)
  • Base URL: https://api.maisonlooks.com
  • Version path: /public/v1

Endpoints

Method Path Notes
GET /public/v1/categories Categories with active products (slug matches site /c/{slug})
GET /public/v1/products Query: category, limit (max 100), offset
GET /public/v1/products/{slug} Single product; 404 if missing
GET /public/v1/outfits Query: featured, limit, offset, productSlug
GET /public/v1/outfits/{id} Single outfit (shortId or UUID)

Example (curl)

curl -H "X-API-Key: $MATRIX_API_KEY" \
  "https://api.maisonlooks.com/public/v1/categories"

Slug aliases (common mistakes)

Avoid Use
shoes sneakers
hoodies-sweaters hoodies-sweatshirts
coats outerwear
tees t-shirts

Validate slugs with GET /public/v1/categories before filtering products. Full handoff (fetch script, types, build notes) is in the repo: docs/maisonlooks-api-handoff.md
